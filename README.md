# 📊 Retail Sales SQL Practice (Google BigQuery)

This project contains a set of SQL queries designed to practice and demonstrate core data analysis skills using Google BigQuery.

The dataset used is a **Retail Sales Dataset**, and the exercises cover fundamental SQL concepts such as filtering, aggregation, grouping, conditional logic, and calculated fields.


## 🚀 Objectives

The goal of this project is to:

- Strengthen SQL querying skills
- Apply real-world data analysis techniques
- Understand how to manipulate and summarize datasets
- Practice writing efficient queries in Google BigQuery


## 📂 Dataset

- **Dataset Name:** Retail Sales Dataset  
- **Platform:** Google BigQuery  
- **Table Name:** `Data` (or your assigned table name)


## 🧠 SQL Concepts Covered

- WHERE Clause
- Filtering with Conditions
- Aggregate Functions (SUM, AVG)
- DISTINCT
- GROUP BY
- CASE Statements
- Conditional Aggregation
- HAVING Clause
- Calculated Fields


## 📝 Exercises Overview

### Q1: WHERE Clause
Filter all transactions that occurred in the year 2023.

### Q2: Filtering + Conditions
Display transactions where Total Amount is greater than the dataset average.

### Q3: Aggregate Functions
Calculate total revenue (SUM of Total Amount).

### Q4: DISTINCT
List all unique product categories.

### Q5: GROUP BY
Calculate total quantity sold per product category.

### Q6: CASE Statement
Classify customers into:
- Youth (<30)
- Adult (30–59)
- Senior (60+)

### Q7: Conditional Aggregation
Count high-value transactions (Total Amount > 500) per gender.

### Q8: HAVING Clause
Show product categories with total revenue greater than 5,000.

### Q9: Calculated Fields
Classify transactions based on Price per Unit:
- Cheap (<50)
- Moderate (50–200)
- Expensive (>200)

### Q10: WHERE + CASE
Filter customers aged 40+ and classify spending:
- High (Total Amount > 1000)
- Low (otherwise)
